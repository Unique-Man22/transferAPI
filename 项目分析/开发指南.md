## 部署指南

### 1. 生产环境准备
```bash
# 安装生产依赖
pip install -r requirements.txt

# 配置生产环境变量
export ENV=production
export VOLC_API_KEY=your-production-key
export LOCAL_API_KEY=strong-random-key

# 创建服务配置文件
sudo nano /etc/systemd/system/doubao-transfer.service
```

### 2. Systemd 服务配置
```ini
[Unit]
Description=DouBao Transfer API Service
After=network.target

[Service]
Type=simple
User=www-data
Group=www-data
WorkingDirectory=/path/to/project
Environment="ENV=production"
Environment="VOLC_API_KEY=your-key"
Environment="LOCAL_API_KEY=your-local-key"
ExecStart=/usr/bin/python3 main.py
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

### 3. Nginx 反向代理配置
```nginx
server {
    listen 80;
    server_name api.yourdomain.com;
    
    location / {
        proxy_pass http://127.0.0.1:5187;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # 超时设置
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 300s;
    }
    
    # 限制请求体大小
    client_max_body_size 10M;
}
```

### 4. Docker 部署
```dockerfile
# Dockerfile
FROM python:3.9-slim

WORKDIR /app

# 安装依赖
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# 复制代码
COPY . .

# 创建非root用户
RUN useradd -m -u 1000 appuser && chown -R appuser:appuser /app
USER appuser

# 暴露端口
EXPOSE 5187

# 启动命令
CMD ["python", "main.py"]
```


## 贡献指南

### 1. 代码提交规范
```bash
# 创建功能分支
git checkout -b feature/new-feature

# 提交代码
git add .
git commit -m "feat: 添加新功能说明

- 详细说明变更内容
- 修复的问题
- 测试情况"

# 推送到远程
git push origin feature/new-feature
```

### 2. 提交信息格式
```
类型(范围): 简短描述

详细描述：
- 变更内容1
- 变更内容2
- 测试情况

关联Issue: #123
```

**类型说明**：
- `feat`: 新功能
- `fix`: 修复bug
- `docs`: 文档更新
- `style`: 代码格式调整
- `refactor`: 代码重构
- `test`: 测试相关
- `chore`: 构建过程或辅助工具变更

### 3. 代码审查要点
1. **功能正确性**：新功能是否按预期工作
2. **代码质量**：是否符合编码规范
3. **测试覆盖**：是否有相应的测试用例
4. **文档更新**：是否更新了相关文档
5. **向后兼容**：是否影响现有功能

### 4. 发布流程
1. **功能开发** → 2. **代码审查** → 3. **测试验证** → 4. **文档更新** → 5. **版本发布**
